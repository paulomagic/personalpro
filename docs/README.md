# Documentação do Projeto

Este diretório concentra apenas documentação ativa.

## Guias principais

- `APP_CURRENT_STATE.md`
  - visão real do app hoje: arquitetura, papéis, módulos, limites e pontos pendentes
- `LGPD_OPERATIONS.md`
  - operação de privacidade, migrations, validação e escopo atual
- `PWA_OFFLINE.md`
  - comportamento offline, limitações e roteiro de teste
- `PUSH_SETUP.md`
  - setup de web push com Supabase e service worker
- `SECRETS_POLICY.md`
  - política de segredos, chaves e dados sensíveis
- `adr/0001-architecture-v2.md`
  - decisão arquitetural vigente

## Documentos na raiz

- `README.md`
  - entrada principal do repositório
- `SECURITY_CHECKLIST.md`
  - checklist operacional de segurança
- `INCIDENT_RESPONSE.md`
  - resposta a incidentes

## Política desta pasta

- manter apenas documentação que reflita o código atual
- remover material histórico que já não ajude na operação
- evitar changelog manual ou “auto-update” sem curadoria
- preferir poucos documentos canônicos, com responsabilidade clara
