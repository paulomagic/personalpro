# Habilidade de Gerenciamento de Monorepo (Agente Original: monorepo-architect)

## Capacidades

- Seleção de ferramentas de monorepo (Nx, Turborepo, Bazel, Lerna)
- Configuração de workspace e estrutura de projeto
- Cache de build (local e remoto)
- Gerenciamento de grafo de dependências
- Detecção de arquivos afetados/alterados para otimização de CI
- Compartilhamento de código e extração de bibliotecas
- Orquestração e paralelização de tarefas

## Quando Usar

- Configurar um novo monorepo do zero
- Migrar de polyrepo para monorepo
- Otimizar pipelines de CI/CD lentos
- Compartilhar código entre múltiplas aplicações
- Gerenciar dependências entre projetos
- Implementar ferramentas consistentes entre equipes

## Fluxo de Trabalho

1. Avaliar o tamanho da base de código e estrutura da equipe
2. Selecionar a ferramenta de monorepo apropriada
3. Projetar estrutura de workspace e projeto
4. Configurar estratégia de cache de build
5. Configurar detecção de afetados/alterados
6. Implementar pipelines de tarefas
7. Configurar cache remoto para CI
8. Documentar convenções e fluxos de trabalho

## Melhores Práticas

- Comece com limites de projeto claros
- Use convenções de nomenclatura consistentes
- Implemente cache remoto cedo
- Mantenha bibliotecas compartilhadas focadas
- Use tags para restrições de dependência
- Automatize atualizações de dependência
- Documente o grafo de dependência
- Estabeleça regras de propriedade de código
